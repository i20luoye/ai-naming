# AI起名工具 — UI/UX 设计方案

> 版本：v1.0 | 2026年6月
> 设计定位：AI起名工具（传统文化科普 × 现代科技体验）

---

## 一、设计定位

### 1.1 产品气质定义

**一句话定位**：一款"有学问但不卖弄、有文化但不陈腐、有科技但不冰冷"的AI起名工具。

| 维度 | 要 | 不要 |
|------|-----|------|
| 文化感 | 典雅、温润、有底蕴 | 陈旧、土气、封建感 |
| 科技感 | 简洁、高效、智能 | 冷冰冰、机械、过度炫技 |
| 情感调性 | 温暖、庄重、有仪式感 | 迷信、恐吓、贩卖焦虑 |
| 信息表达 | 专业术语 + 人话翻译 | 堆砌术语吓唬用户 |
| 视觉联想 | 一本精美的古籍 + 一个现代App | 街边算命摊、风水罗盘 |

**核心设计原则**：
1. **翻译原则**：所有八字/五行专业术语必须配"人话翻译"，降低认知门槛
2. **仪式感原则**：从输入到拿到名字的流程像"打开一份精心准备的信笺"，不是"点了个外卖"
3. **克制原则**：信息量大但不堆砌，每个页面只讲一件事
4. **透明原则**：为什么推荐这个名字？解释清楚逻辑，不故作神秘

### 1.2 目标用户画像

- **主力用户**：25-35岁新手父母，女性占比高
- **使用场景**：孕期/产后在家用手机刷小程序、睡前和伴侣一起挑名字
- **核心诉求**：想要一个"不烂大街、有文化出处、五行也匹配"的好名字
- **痛点**：自己翻字典太累、网上名字同质化严重、不懂八字怕踩坑
- **付费心理**："怕后悔"——名字跟一辈子，不能随便

## 三、核心页面设计

### 3.1 首页 / 输入页

#### 设计目标
降低输入门槛，让用户3分钟内完成信息填写。视觉上营造"郑重但不沉重"的氛围。

#### 线框描述

```
┌─────────────────────────────────┐
│         [Logo]  AI起名           │  ← 品牌区，简洁
│      以字为礼，以名为印           │  ← Slogan，小字
├─────────────────────────────────┤
│                                 │
│  ● 宝宝的出生日期               │
│  ┌─────────────────────────┐   │
│  │ 📅 请选择出生日期    〉  │   │  ← 日期选择器（触发picker）
│  └─────────────────────────┘   │
│                                 │
│  ● 出生时间                     │
│  ┌─────────────────────────┐   │
│  │ 🕐 请选择出生时辰    〉  │   │  ← 时辰选择器（12时辰+未知）
│  │    如不确定可先跳过       │   │  ← 友好提示
│  └─────────────────────────┘   │
│                                 │
│  ● 出生地点                     │
│  ┌─────────────────────────┐   │
│  │ 📍 搜索城市名称      🔍 │   │  ← 搜索型输入，用于真太阳时

│                                 │

│  ┌──────────┐ ┌──────────┐    │

│  └──────────┘ └──────────┘    │
```

#### 设计要点

| 要点 | 方案 |
|------|------|
| **日期选择** | 微信原生日期选择器（picker-view），默认选中当前日期前推1年的范围 |
| **时辰选择** | 12时辰滚轮选择器，标注对应时间范围（如"子时 23:00-01:00"），底部提供"不确定，先跳过"链接 |
| **出生地搜索** | 输入城市名后实时匹配，列表展示"北京（东经116.4°）"，解释为什么需要地点 |
| **性别选择** | 大按钮而非小radio，触控区域≥88×88px（符合微信小程序规范） |
| **姓氏输入** | 自动聚焦，限制1-2个汉字，输入后即时校验（是否为常见姓氏、是否含非汉字字符） |
| **按钮状态** | 必填项未完成时按钮半透明+不可点击；全部完成后按钮出现微动效（呼吸光晕）引导点击 |
| **合规声明** | 页面底部固定，灰色小字，不可隐藏："内容仅为传统文化民俗参考，不构成人生决策依据" |

#### 设计参考
- 微信原生表单页面的简洁风格
- Apple Health 的信息录入流程（分步引导、大触控区域）

| 要点 | 方案 |
|------|------|
| **四柱展示** | 4列卡片式，天干用大号楷体（有书法感），地支中号，五行用小圆点标签 |
| **五行雷达图** | 用canvas绘制五边形雷达图，5个顶点分别对应五行，填充区域颜色对应五行色；旁边配横向进度条辅助阅读 |
| **日主卡片** | 核心信息突出：日主天干+喜用神用色块标注，"人话翻译"区域用引号+柔和背景色区分 |
| **加载动效** | 八字排盘计算时，展示一个"笔墨书写"动画（约1-2秒），增强仪式感 |
| **分享按钮** | 右上角提供分享按钮，生成一张精美的"宝宝命盘卡片"可分享到微信 |

#### 关键设计决策
- **不在此页展示大运/流年/十神**等专业信息——这不是算命工具。用户目标是起名，只需要知道五行缺什么、喜用什么
- 如果用户对八字不感兴趣，可提供"跳过分析，直接看名字"的快捷入口

#### 卡片详细设计

每个名字卡片包含以下信息层级：

```
第一层（视觉重心）：名字大字 + 五行标签
第二层（快速判断）：寓意一句话 + 综合评分
第三层（深度信息）：拼音、音韵、笔画、出处
第四层（行动按钮）：收藏
```

#### 五行标签设计

采用**圆角小标签**，每个标签包含五行图标+五行名称，颜色映射：

| 五行 | 图标 | 背景色 | 文字色 |
|------|------|--------|--------|
| 木 | 🪵/🌿 | #E8F5E9 | #2E7D32 |
| 火 | 🔥/☀️ | #FCE4EC | #C62828 |
| 土 | 🏔️/⛰️ | #FFF3E0 | #E65100 |
| 金 | ⚪/💎 | #F5F5F5 | #616161 |
| 水 | 💧/🌊 | #E3F2FD | #1565C0 |

#### 设计要点

| 要点 | 方案 |
|------|------|
| **卡片间距** | 卡片间距16px，上下各8px留白，确保每屏能看到1.5-2张卡片，引导下滑 |
| **评分展示** | 用星级+数字组合，满分100。90分以上用金色，80-89用银色，80以下灰色 |
| **收藏交互** | 点击心形图标，心形填充动画（约300ms），同时卡片右上角出现一个小书签角标 |
| **筛选功能** | 横向滚动chips风格（类似小红书标签），选中态高亮。风格：古风/简约/大气/诗意/现代；字数：单字/双字/不限；五行：按需勾选 |
| **加载策略** | 首次展示8个名字，下滑自动加载更多（每次5个），总共推荐15-30个 |
| **空状态** | 如果筛选后无结果，展示"换个筛选条件试试" + 推荐重置按钮 |
| **换方向** | 底部提供"换一批"按钮，重新调用AI生成不同方向的名字 |

#### 微动效
- 卡片入场：从下往上淡入，每张间隔80ms（staggered animation）
- 收藏动效：心形从小到大弹性动画 + 轻微震动反馈（haptic）
- 评分数字：从0滚动到目标分数（约500ms）

| **分享** | 使用微信原生分享按钮`<button open-type="share">` |
| **收藏同步** | 使用微信Storage缓存收藏数据，未登录也可使用 |

| 要点 | 方案 |
|------|------|
| **响应式** | 最大宽度480px居中，两侧留白，大于768px时展示双栏布局 |
| **字体** | 使用rem单位，根字号16px，基于375px设计稿等比缩放 |
| **触摸** | 禁用iOS双击缩放、300ms点击延迟 |
| **分享** | 使用Web Share API + 降级方案（生成图片长按保存） |

## 七、设计参考

### 7.1 竞品设计分析

#### 1. 千千起名

| 维度 | 评价 |
|------|------|
| **优点** | 界面简洁清爽无广告，功能分区清晰，操作步骤简单，老人也能上手。核心功能免费，无强制付费套路。名字库源自诗词经典，每个名字有详细解析 |
| **缺点** | 视觉风格偏传统，缺少现代设计感；色彩偏暗沉；卡片信息层级不够清晰；整体气质偏"工具"而非"体验"，缺少情感化设计 |
| **借鉴** | 简洁的功能分区、免费+增值的商业模式、名字附带完整解析 |

#### 2. 宝宝起名App / 起名神器小程序

| 维度 | 评价 |
|------|------|
| **优点** | 轻量化、操作简单、加载快。内置多种起名模式，支持多维度筛选 |
| **缺点** | 广告多，体验被广告打断；设计粗糙，缺乏品质感；八字排盘准确性存疑；名字同质化严重 |
| **借鉴** | 轻量化设计思路、多维筛选交互 |

#### 3. 元小满

| 维度 | 评价 |
|------|------|
| **优点** | 差异化定位"国学+心理"，走精细化小众路线。界面设计清新生动，国风壁纸等特色内容提升用户粘性。隐私保护意识强 |
| **缺点** | 功能深度不足，起名能力弱于专业工具；用户规模小，品牌认知度低 |
| **借鉴** | 国风+现代的设计融合、隐私保护机制、内容化运营思路 |

#### 4. 易奇文化

| 维度 | 评价 |
|------|------|
| **优点** | 专业度高，八字排盘功能完善，支持多种排盘版式；用户基数大（1000万+） |
| **缺点** | 功能入口多且杂，新手学习成本高；界面偏"工具型"缺少设计感；整体气质偏"算命"而非"文化" |
| **借鉴** | 专业排盘功能的设计参考、多种版式切换思路 |

#### 5. NameHatch（海外参考）

| 维度 | 评价 |
|------|------|
| **优点** | Tinder式滑动交互极其简单有趣，伴侣协同筛选机制设计精妙，UI极简现代 |
| **缺点** | 完全不涉及八字/五行，文化内涵零；仅适合英文名 |
| **借鉴** | 滑动交互的趣味性、伴侣协作的设计模式（可以考虑"分享给配偶一起选"功能）、极简UI风格 |
