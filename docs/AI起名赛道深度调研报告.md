### 4.2 实现路径

**方案A：自研八字引擎 + LLM增强（推荐）**
- 八字排盘部分：纯算法，确定性高，可自研（Python即可实现）
- 喜用神判断：规则引擎 + 知识图谱
- 名字生成：调用LLM（GPT/文心/DeepSeek），用结构化prompt约束五行、音韵、寓意
- 优点：核心命理逻辑可控，LLM只负责创意部分
- 开发量：中等（2-4周MVP）

**方案B：调用第三方API + 自建体验层**
- 八字起名API已可用：gugudata API（十年模型积累，三才五格+喜用神）、apizero API
- 优点：快速上线，命理部分外包
- 缺点：命理逻辑黑箱，难以差异化

**方案C：纯LLM方案**
- 直接用GPT-4/DeepSeek等大模型完成八字→名字全流程
- 优点：最快
- 缺点：LLM做八字计算不可靠（数学能力弱），需要大量prompt engineering，容易出错

**建议：方案A为主 + 方案B为备选。** 核心命理引擎自研确保准确性，名字创意部分用LLM增强。

### 4.3 关键技术难点

1. **真太阳时校准** — 需要经纬度换算，不能只用北京时间
2. **喜用神判断** — 不是"缺啥补啥"，需要扶抑/调候/通关综合分析，这是80%竞品做错的地方
3. **方言谐音检测** — 粤语、闽南语、客家话等方言谐音库需要建设
4. **重名率数据** — 需要对接公安部或第三方姓名数据库
5. **LLM幻觉控制** — 让LLM生成的名字严格符合五行要求需要精心设计的prompt

## 七、风险与挑战

4. **数据获取** — 重名率数据、方言谐音数据需要持续积累。
5. **LLM成本** — 如果用大模型生成名字，API调用成本需要控制。
