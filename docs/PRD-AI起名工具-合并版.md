## 一、产品概述

### 1.1 一句话定位

**一款以「八字五行精准排盘 + 汉字文化深度匹配」为核心、AI大模型增强体验的智能起名工具。**
定位为「AI起名工具」（文化解读 + 音形义筛选），**不是「AI算命」**。

### 1.2 核心差异化

| 竞品痛点 | 我们的解决方案 |
|---------|-------------|
| 排盘不准（大模型直接排盘错误率40%） | **规则引擎排盘**，lunar-javascript保证100%准确 |
| 「缺啥补啥」过于机械 | **五行旺衰 + 喜用神推断**，精准定位真正需要补的五行 |
| 名字同质化严重（梓涵、一诺泛滥） | **60,000条康熙字典数据库** + AI文化释义，生成差异化名字 |
| 解读浅层，缺乏文化深度 | **大模型深度解读**：诗词典故、字形字义、音韵分析 |
| 海外华人无法享受专业服务 | **真太阳时校正** + 全线上服务，覆盖全球时区 |

### 1.3 平台策略

- MVP：微信小程序（获客主阵地）
- 同步：H5 Web端（Next.js，覆盖海外用户+SEO）
- 后续：原生App（Flutter，视用户增长决定）

## 四、核心业务流程

### 4.1 用户旅程

```
首页 → 信息输入（公历/农历+时间+地点+姓氏+性别）
  → 八字排盘（真太阳时校正→四柱→十神→神煞）
  → 五行分析（日主强弱→旺衰判定→喜用神推断）
  → AI名字生成（喜用神筛选字库→LLM生成→合规过滤→质量评估）
  → 结果展示（免费3个/付费全部→名字详情→收藏/分享）
```

### 4.2 数据流架构

```
前端(小程序/H5) → API Routes(Next.js)
  ├── 排盘引擎(lunar-javascript + bazi-calculator) → 纯规则，<200ms
  ├── 喜用神引擎(自建TS，参考china-testing/bazi) → 规则+评分
  ├── 字库服务(2万汉字DB + cnchar) → PostgreSQL查询
  ├── AI服务(DeepSeek V3 + GPT-4o-mini备选) → SSE流式输出
  ├── 合规过滤层(规则引擎) → 宁阻勿漏
  └── 名字质量评估(五行补益+三才五格+谐音+热名) → 综合评分
```

## 六、算法逻辑说明

### 6.1 八字排盘
- 输入：出生年月日时+出生地经纬度
- 处理：公历→农历→真太阳时校正→四柱计算→十神→神煞
- 输出：四柱八字+五行分布+日主天干

### 6.2 喜用神判定（核心差异化）
- 输入：四柱八字+五行分布
- 处理：日主强弱评分(得令30%+得地25%+得生25%+得助20%) → 格局判断 → 用神选取(身强克泄耗/身弱生助/调候/通关)
- 输出：喜神/用神/忌神/仇神/闲神 + 置信度 + 判定方法描述

### 6.3 AI名字生成
- 输入：喜用神+姓氏+性别+风格偏好
- 处理：规则引擎筛选候选字 → LLM生成名字 → 合规过滤 → 名字质量评估
- 输出：10+精选名字+完整解析
