# AI起名工具 — 合规与风险控制方案

> 编制时间：2026年6月12日  
> 适用范围：基于生辰八字的AI起名工具（八字排盘 + 五行分析 + 汉字推荐）  
> 核心依据：2026年4月30日网信办"清朗·整治AI应用乱象"专项行动、《生成式人工智能服务管理暂行办法》《个人信息保护法》

## 一、政策红线清单

### 1.1 核心政策背景

2026年4月30日，中央网信办部署为期4个月（至8月底）的"清朗·整治AI应用乱象"专项行动，分两个阶段开展。第二阶段明确将**"提供AI算命等AI违规功能服务"**列为重点打击对象之一（来源：西藏网信办转载工人日报报道，2026年5月1日）。2026年6月，中央网信办举报中心进一步开设"涉AI应用乱象举报专区"，专项受理14类问题举报（来源：央视网/光明网，2026年6月12日）。

### 1.2 不可触碰的红线（逐条列明）

| 序号 | 红线内容 | 具体表现 | 风险等级 |
|------|---------|---------|---------|
| 1 | **AI算命/占卜** | 任何以AI名义进行命运预测、吉凶判断、运势推算的功能 | 🔴 致命 |
| 2 | **改运消灾话术** | 宣传或暗示可通过改名"改运""消灾""避祸""化解厄运" | 🔴 致命 |
| 3 | **诱导大额消费** | 以"命中缺某五行会破财/生病"等恐吓性话术诱导购买高价服务 | 🔴 致命 |
| 4 | **兜售祈福产品** | 以AI名义售卖开光法器、护身符、转运饰品等 | 🔴 致命 |

| 7 | **AI生成内容无标识** | 未按《人工智能生成合成内容标识办法》添加显式+隐式标识 | 🟠 高危 |

## 二、合规定位策略

### 2.1 产品定位话术体系

**核心原则**：将产品从"AI算命"重新定义为"传统文化科普 + 姓名学辅助工具"。

#### 对外宣传标准话术（可直接使用）

> **一句话定位**：「XX起名」是一款基于传统八字命理学的AI姓名推荐工具，融合《康熙字典》汉字五行体系和现代音韵美学，为家长提供**文化参考性质**的宝宝名字方案。所有分析结果仅供传统文化民俗参考，不构成人生决策依据。

> **品牌故事方向**：强调"传承中华姓名文化"、"让每个名字都有文化底蕴"、"八字是一门古老的时间记录科学，五行是中国古人认识世界的哲学框架"——始终将八字五行框定在"传统文化""古典哲学"范畴内，绝不使用"算命""预测""命运"等词汇。

#### 各场景禁用词 & 替代词对照表

| 禁用词（迷信话术） | 替代词（合规表述） |
|------------------|------------------|
| AI算命 | AI八字排盘 / AI五行分析 |
| 改运 / 转运 | （不使用，直接删除此功能） |
| 命中注定 | 传统文化认为 / 古典命理学中 |
| 大富大贵 / 一生顺遂 | 寓意美好 / 寄托期望 |
| 破财 / 血光之灾 | （严格禁止，不输出任何负面预测） |
| 消灾 / 化解 | （严格禁止） |
| 大师 / 道长 / 仙师 | 命名顾问 / 姓名学专家 |
| 开光 / 加持 / 灵验 | （严格禁止） |
| 运势 / 流年吉凶 | 性格倾向分析 / 天赋特质解读 |

### 2.2 功能边界：可以做 vs 绝对不能做

#### ✅ 可以做的功能（合规白名单）

| 功能模块 | 具体内容 | 合规理由 |
|---------|---------|---------|
| **八字排盘** | 根据出生时间+地点，用规则引擎计算四柱八字 | 属于传统历法计算，等同公历转农历工具 |
| **五行旺衰分析** | 统计八字中各五行（金木水火土）的数量分布 | 属于数据分析/统计展示 |
| **喜用神推断** | 基于传统子平术规则，推断命理喜用神 | 标注"传统命理学认为"，属于文化知识科普 |
| **名字五行匹配** | 根据喜用神推荐对应五行属性的汉字 | 属于汉字属性检索+推荐 |
| **三才五格计算** | 计算天格/人格/地格/外格/总格的数理 | 属于传统姓名学算法 |
| **音韵平仄检测** | 检测名字的声调搭配、押韵情况 | 属于语言学工具 |
| **诗词典故引用** | 说明名字用字的古诗文出处 | 属于文化知识科普 |
| **重名度查询** | 查询名字在全国的重名概率 | 属于公开数据查询 |
| **生肖宜忌参考** | 根据生肖推荐/规避特定偏旁部首 | 标注"传统民俗认为" |

#### ❌ 绝对不能做的功能（违规黑名单）

1. **运势预测**：任何关于未来财运、事业运、感情运、健康运的预测
2. **吉凶断言**：宣称"这个名字能让孩子一生富贵""这个名字克父母"
3. **改运建议**：建议用户因为"命不好"而改名字、改风水
4. **择吉日**：推荐"吉日"进行剖腹产、结婚、开业等
5. **命理咨询对话**：AI自由对话式命理问答（极易越界）
6. **转运商品**：任何形式的祈福产品、护身符、法事服务推荐
7. **八字合婚**：基于八字的婚姻匹配/吉凶判断
8. **面相/手相分析**：上传照片进行面相分析

| 维度 | 错误定位 | 正确定位 |

| 功能描述 | AI帮你算一个好命的名字 | AI帮你选一个寓意美好的名字 |
| 目标用户 | 想改运的人 | 想给孩子取个好名字的家长 |

## 三、内容安全方案

### 3.1 敏感词过滤规则

#### 3.1.1 违禁词黑名单（一级拦截 — 直接阻断输出）

以下词汇及其变体在AI输出中**完全禁止出现**：

```
【命运断言类】
命中注定、天意、命里、宿命、定数、前世今生、因果报应、报应、轮回

【鬼神宗教类】
神佛、菩萨、佛祖、上帝、鬼神、鬼怪、妖魔、精灵、灵异、显灵、开光、加持

【改运消灾类】
改运、转运、消灾、化解、破解、避祸、挡煞、辟邪、驱邪、禳解、祈福消灾

【负面预测类】
破财、血光、灾祸、大凶、劫难、厄运、克夫、克妻、克父母、短命、夭折

【诱导消费类】
必须改名否则……、不化解就会……、不请大师不行、花大钱才能……
```

#### 3.1.2 高危词灰名单（二级拦截 — 触发替换或标注）

以下词汇需在上下文中审查，若涉及预测/断言则拦截，若属文化科普则放行：

```
【需审查语境类】
吉凶、旺衰、好坏、顺遂、坎坷、贵人、小人、桃花、财运、事业运
→ 规则：这些词只能出现在"传统命理学认为""古典文化中"等科普语境中，
  绝不能以AI的直接口吻做出断言
```

#### 3.1.3 过滤规则实施

```
实施方式：三层过滤架构
  第一层：关键词硬过滤（输出前，基于黑名单直接删除/替换）
  第二层：语义审核（基于规则+分类模型，检测宿命论/预测性表述）
  第三层：人工抽检（每日随机抽检5%的输出结果，重点检查边缘案例）

过滤结果处理：
  - 命中一级黑名单 → 阻断输出，返回标准拒答话术
  - 命中二级灰名单 → 自动替换为合规表述，记录日志供审核
```

## 四、用户数据合规

### 4.1 生辰八字的数据分类

根据《个人信息保护法》，**出生日期+精确时间+出生地点**（可用于推算生辰八字）属于**敏感个人信息**范畴。理由如下：

1. 生辰八字由出生年月日时推算得出，精确到时辰，能唯一或高度关联到特定自然人
2. 结合姓名等其他信息，可识别个人身份
3. 一旦泄露或滥用，可能导致歧视性待遇（如因"八字不好"被拒绝某些机会）
4. 参考国家标准GB/T 45574-2025《数据安全技术 敏感个人信息处理安全要求》中对特定身份信息的界定

### 4.2 数据合规要求清单

| 要求 | 具体措施 | 对应法规 |
|------|---------|---------|
| **知情同意** | 收集生辰信息前弹窗明确告知：收集目的（仅用于八字排盘）、使用范围（仅用于本次起名）、存储期限、用户权利 | 《个保法》第13-17条 |
| **单独同意** | 生辰信息作为敏感个人信息，需用户**单独勾选同意**（不能混在用户协议中一并勾选） | 《个保法》第29条 |
| **最小必要** | 仅收集起名必需的信息：出生日期、出生时间（可精确到时辰）、出生地点（地级市级别即可，不需要精确地址）、姓氏 | 《个保法》第6条 |
| **加密存储** | 所有生辰数据在数据库中使用AES-256或国密SM4加密存储；传输使用TLS 1.2+ | 《数据安全法》第27条 |
| **数据删除** | 提供"一键删除"功能：用户可随时删除个人所有数据（生辰信息+起名记录），删除后7个工作日内完成所有备份清除 | 《个保法》第47条 |
| **数据不出境** | 所有服务器部署在中国境内，生辰数据禁止跨境传输 | 《个保法》第38-40条 |
| **禁止倒卖** | 严禁将用户生辰数据用于任何第三方营销、出售、共享 | 《个保法》第23条 |
| **未成年人保护** | 收集14岁以下未成年人信息需监护人单独同意；起名场景中新生儿信息属于未成年人信息，需由父母代为同意 | 《个保法》第31条 |

### 4.3 隐私政策关键条款示例

```
【隐私政策 — 敏感个人信息处理说明】

我们收集以下敏感个人信息：
- 出生日期、出生时间、出生地点（用于计算传统八字命盘）
- 姓氏（用于生成候选名字）
```
