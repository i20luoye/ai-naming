> **版本**：v1.0 | **日期**：2026年6月 | **密级**：内部使用
>
> **项目代号**：AI起名工具 | **负责人**：大明
>
> **核心判断**：2026年4月30日中央网信办"清朗·整治AI应用乱象"专项行动明确将"AI算命"列为打击对象，合规是该项目**首要生存问题**。但"传统文化科普+AI辅助"定位存在合规路径，前提是产品设计、用词、功能边界严格遵守本报告规范。

## 一、摘要

本报告针对"AI起名工具"项目，全面评估其面临的合规风险，并提供可执行的合规设计方案。

**核心结论**：

| 项目 | 评估 |
|------|------|
| 能否做 | ✅ 可以，但必须严格合规 |
| 最大风险 | 🔴 被认定为"AI算命"而遭下架/处罚 |
| 合规关键 | 定位"传统文化科普工具" + 完成算法备案 + AI内容标识 + 严格内容审核 |
| 时间窗口 | 建议立即启动算法备案（周期4-6个月），在清朗行动结束（8月底）前完成合规准备 |
| 违规后果 | 产品下架、罚款（最高1000万元）、刑事责任 |

## 三、法律边界分析

### 3.1 《互联网信息服务管理办法》第十五条

> **原文**：互联网信息服务提供者不得制作、复制、发布、传播含有下列内容的信息：……（五）宣扬邪教和**封建迷信**的……

**法律解读**：
- "封建迷信"的法律定义在实践中较为宽泛
- 关键区分：**传统文化知识科普** vs **封建迷信活动**
- 司法实践中的判断标准：
  - ❌ 宣称能"预测吉凶""改变命运""消灾避祸"→ 封建迷信
  - ✅ 介绍五行八字作为传统文化知识 → 科普
  - ❌ 暗示名字能"决定命运""带来好运"→ 迷信
  - ✅ 分析名字的音韵美感、文化内涵 → 文化服务

### 3.2 《生成式人工智能服务管理暂行办法》第四条

> 提供和使用生成式人工智能服务，应当遵守法律、行政法规，尊重社会公德和伦理，**遵守社会主义核心价值观**……

**对本项目的影响**：
- AI生成的名字解读必须过滤"命定论""宿命论"内容
- 不得输出"此名必大富大贵""此名可改运"等表述
- 解读应限定在：性格分析、文化内涵、音律美感、五行搭配的传统文化视角

### 3.3 关键法律红线与合规空间

| 行为 | 定性 | 风险等级 |
|------|------|:---:|
| 宣称"AI算命""预测命运" | 违规 | 🔴 |
| 提供八字排盘（工具属性） | 灰色/合规 | 🟡 |
| 基于八字推荐名字（文化参考） | 灰色/合规 | 🟡 |
| 宣称名字能"改运""消灾" | 违规 | 🔴 |
| 提供"传统文化科普+AI辅助起名" | 可合规 | 🟢 |
| 售卖"开光""转运"产品 | 违规 | 🔴 |
| 提供名字打分、重名查询 | 合规 | 🟢 |

### 5.2 功能边界

```
✅ 可以做的功能：
├── 八字排盘（工具属性，类似万年历）
├── 五行分析（传统文化知识科普）
├── 名字推荐（基于音韵、寓意、文化内涵）
├── 名字打分/重名查询
├── 诗经/楚辞名字出处查询
├── 生肖用字宜忌（民俗文化参考）

❌ 不能做的功能：
├── "命运预测"功能
├── "吉凶判断"功能
├── "运势分析"功能
├── "改运建议"功能
├── "AI算命"入口
├── 付费"开光"服务
└── 任何形式的"转运"产品推荐
```

### 5.3 AI生成内容标识

根据《人工智能生成合成内容标识办法》（2025.9.1施行）及配套强制性标准：

1. **显式标识**：所有AI生成的名字报告必须在显著位置标注"**本内容由AI生成，仅供参考**"
2. **隐式标识**：在生成内容的元数据中嵌入AI生成标记
3. **标识规范**：
   - 标识大小：不小于正文字号
   - 标识位置：报告首部和尾部均需出现
   - 标识颜色：与正文有足够对比度
4. **不得提供"去标识"功能或教程**

### 5.4 免责声明模板

```
【必加声明 — 报告首部】
本工具基于中国传统文化中的生辰八字、五行学说及姓名学理论，结合AI技术
提供起名参考建议。所有分析和推荐仅供传统文化爱好者和用户参考，不构成
人生决策依据，不具备命运预测或吉凶判断效力。

【必加声明 — 报告尾部】
声明：本报告由AI生成，内容仅为传统文化民俗参考。起名是家庭的重要决策，
建议结合家庭传统、个人喜好和实际情况综合考量。本工具不提供"算命"、
"改运"、"预测"等服务，请理性看待分析结果。

【用户协议中必含】
使用本工具即表示您理解并同意：本工具提供的是传统文化科普和起名参考服务，
并非"算命"或"命运预测"工具。AI生成的所有内容仅供娱乐和文化参考，
不应作为人生重大决策的依据。
```

### 5.5 内容审核机制

建议建立三层过滤机制：

| 层级 | 机制 | 负责 |
|------|------|------|
| L1 输入过滤 | 敏感词/意图识别，拦截"算命""改运"等请求 | 算法自动 |
| L2 输出过滤 | 实时审核AI生成内容，过滤命定论/宿命论/鬼神论词汇 | AI+规则引擎 |
| L3 人工抽检 | 每日抽检生成内容，持续优化过滤规则 | 运营团队 |

**必须过滤的输出关键词**（示例）：
```
命中注定、大富大贵、血光之灾、破财、改运、转运、消灾、避祸、
天机不可泄露、因果报应、前世今生、鬼神、妖魔、附体、作法、
开光、加持、灵验、包准、百分百、必定、注定
```

### 6.3 本项目备案策略

**推荐路径：大模型登记 + 算法备案**

- 若仅调用已备案大模型API（如DeepSeek/通义千问），做简单Prompt工程，不微调模型：
  - → **大模型登记**（1-2月）+ **算法备案**（2-3月）
- 若自研/微调模型：
  - → **大模型备案**（4-6月）+ **算法备案**（2-3月）

**建议**：初期优先使用API调用模式，降低备案复杂度。

### 6.4 备案避坑要点

1. **备案以"算法"为单位，非"产品"**：起名生成算法、八字排盘算法等需分别备案
2. **"应备未备"是清朗行动第一类打击对象**
3. **材料务必真实**：提供虚假材料将被撤销备案+处罚
4. **备案后义务**：显著位置展示编号；算法变更10日内更新；保留日志≥6个月
5. **安全评估报告是关键难点**：建议聘请专业机构协助

## 七、数据隐私与安全

### 7.1 敏感信息界定

根据《个人信息保护法》：
- **出生年月日时 + 出生地点** → 属于"敏感个人信息"（与个人身份强关联，可用于精准画像）
- 收集前必须：
  - ✅ 取得用户**单独同意**（不能混在隐私政策里）
  - ✅ 告知收集目的、使用方式、存储期限
  - ✅ 提供撤回同意的途径

### 7.2 数据安全措施

| 措施 | 说明 |
|------|------|
| 加密存储 | 生辰数据AES-256加密，密钥与数据分离 |
| 最小化原则 | 只收集起名必需的出生时间+性别，不收集姓名、身份证号 |
