## 三、🔥 激活转化层（P1 — 来了要能留住）

### 4. 首页 Landing Page 转化力不足

**现状：** 首页 1233 行，信息量很大但**转化路径模糊**：

- "今日吉名"功能是静态展示，用户参与感为零
- 六道工序是自嗨展示，用户不关心你的工序，关心"结果怎么样"
- 案例区是假数据，缺乏说服力

**修改建议：**
```
Hero 区重构逻辑：
1. 标题：一句话说清楚价值（"AI根据宝宝八字，推荐全网重名率最低的好名字"）
2. 副标题：信任锚（"已为 2,386 个宝宝推荐良名" — 动态计数）
3. CTA 按钮极简：只有一个「免费起名」→ 直接跳输入页
4. 次级 CTA：「测测你名字的分数」→ 跳测名页

砍掉/弱化：
- "今日吉名" → 改为互动式"输入姓氏，看看AI推荐什么名字"（浅试用）
- 六道工序 → 折叠为一行图标+文字，不要占大块面积
- 假案例 → 砍掉或替换为真实用户评价（即使只有几条）

新增：
- 信任符号区：媒体报道 Logo、用户数计数器、评分
```

- 降低决策成本：Step 3（偏好）改为可选，有默认值，不选也能出结果

## 四、💰 付费留存层（P1 — 转化是生存线）

### 7. 付费墙设计缺失

**现状：** 代码中没有付费墙逻辑，所有结果免费可见。这符合 MVP 阶段，但需要提前设计。

**修改建议：**
```
免费版（引流）：
- 八字排盘：完全免费
- AI 起名：展示 3 个名字 + 基础五行分析
- 测名：展示总分 + 一句话评价

付费版（9.9 元）：
- 起名：展示全部 10 个名字 + 详细命理推导链 + 诗词出处
- 测名：五维详细评分 + 改名建议 + 五行补益方案

实现建议：
- 前端展示所有名字卡片，但付费内容用模糊遮罩
- CTA："解锁完整分析 ¥9.9"
- 支付接入：微信支付 + 支付宝
```

### 8. 无裂变机制

**现状：** 无分享激励、无邀请奖励、无社交货币。

**修改建议（低成本高回报）：**
- **结果分享**：生成精美的名字卡片（含五行色彩），用户可保存/分享到朋友圈 → 天然传播
- **家人协作**：起名结果生成"投票链接"，家人可以投票选名字 → 一个用户带来 N 个访问
- **推荐奖励**：分享链接被使用后，双方各得 1 次免费详细报告

### 9. 无用户留存手段

**现状：** 无账号体系、无收藏功能、无历史记录。

**修改建议：**
- 最轻量：用 localStorage 保存用户的起名/测名历史（当前已有 storage.ts，扩展即可）
- 中等：加"收藏名字"功能，收藏夹可分享
- 完整：微信登录 + 用户中心（我的名字、我的排盘、我的收藏）

## 六、技术层面影响增长的点

### 10. 页面加载速度

- `Noto Serif SC` + `Noto Sans SC` CJK 字体体积大，首屏加载慢
- **建议**：字体子集化 + `font-display: swap` + preload
- 页面速度直接影响 SEO 排名和用户跳出率

### 11. 无 PWA / 离线能力

- 起名工具天然适合 PWA（用户可能多次回来查看结果）
- **建议**：加 `manifest.json` + Service Worker，支持"添加到主屏幕"

### 12. 国际化预留

- 增长策略提到海外华人市场是蓝海（$29.99 定价，竞争空白）
- **建议**：即使 MVP 不做英文版，至少把所有中文文案抽到 `src/lib/i18n/`，不要硬编码在 JSX 里
