## 项目名称

- 仓库名：`ai-naming`
- package name：`tianyan-ai-naming`
- 产品名：天衍 AI 起名

## 当前定位

天衍 AI 起名是一个中文 AI 起名 MVP，面向宝宝起名与姓名测评场景。产品以八字排盘、五行分析、诗词出处、音韵与寓意分析作为传统文化参考维度，再通过 AI 生成名字建议或测名评分。

合规定位必须保持为“传统文化参考”“五行分析”“文化民俗参考”。不能表达为算命、改运、预测人生或改变命运。

## 当前已实现内容

- 首页：`src/app/(home)/page.tsx`
  - 品牌展示、起名/测名入口、流程介绍、案例、口碑与底部 CTA。

- AI 起名流程：
  - `/name/input`：采集姓氏、性别、出生日期/时辰、出生地、期望寓意等信息。
  - `/api/bazi`：调用 `src/lib/bazi.ts` 进行八字排盘、五行统计、日主强弱、喜用神与格局推导。
  - `/name/bazi`：展示排盘与五行分析，并缓存排盘结果。
  - `/name/preference`：采集字数、风格、诗词来源、排除字、避热名、避谐音等起名偏好。
  - `/name/result`：调用 `/api/generate-names` 生成名字结果，并展示候选名。
  - `/name/detail`：展示候选名字详情、收藏与选择状态。

- AI 测名流程：
  - `/test-name`：采集姓名及可选出生信息。
  - `/api/test-name`：调用 LLM 返回测名评分、逐字解析、五格、音韵、谐音、重名风险、建议等结构化结果。
  - `/test-name/result`：展示测名结果。

- 后端/API：
  - `/api/bazi`：本地排盘计算，无 LLM 依赖。
  - `/api/generate-names`：使用 OpenAI-compatible LLM API 调用模型 `agnes-2.0-flash`。
  - `/api/test-name`：使用同一模型进行姓名评测。
  - `src/lib/rate-limit.ts`：内存级 IP 限流，适合 MVP，不适合多实例生产环境。
  - `src/lib/compliance.ts`：对 LLM 输出做合规词过滤。

- 前端状态：
  - 主要使用 `localStorage` 在多页面流程间传递输入、排盘、偏好、生成结果和测名结果。
  - 存储键集中在 `src/lib/storage.ts`。

## 当前未实现内容

- 没有用户登录、账号体系、权限体系。
- 没有数据库落库，业务状态主要在浏览器 `localStorage`。
- 没有支付、订单、套餐、权益或消费次数闭环。
- 没有后台管理、内容审核后台、运营配置后台。
- 没有 Redis 或分布式限流；当前限流仅为内存 Map。
- 没有正式的监控、告警、日志分析、埋点分析。
- 没有完整自动化测试体系；当前以 lint、TypeScript 检查和构建为主要验证方式。
- 没有企业起名、公司起名、改名咨询等完整产品流程；首页中出现的相关入口不应视为已实现。
- 没有真实重名数据库、热名数据库或方言谐音数据库接入；相关分析主要依赖 LLM 输出与前端展示。
