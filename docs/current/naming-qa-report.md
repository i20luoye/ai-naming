# Real Naming QA Report

> 状态：Real Naming QA Pack v1  
> 日期：2026-06-20  
> 范围：基于当前 `/api/generate-names` 返回结构、知识层候选、validate、score、fallback 标记做离线 QA 复盘。本文不代表线上大规模真实用户测试，也不导入外部古籍库。

## 测试目标

验证当前 AI 起名链路在典型输入下是否具备可上线前继续迭代的基础质量：

- 能按姓氏、性别、喜用方向、出处偏好、风格偏好和避用字形成可展示结果。
- 返回字段保持向后兼容，并包含 `knowledgeBacked`、`validationWarnings`、`qualityScore`、`qualityLevel`、`riskLevel`。
- 知识层能对名字进行筛选、排序、降级、解释，而不只是事后校验。
- fallback 场景能被明确标记，不混同为知识库校验结果。

## 测试范围

本轮覆盖 36 组典型输入：

- 常见姓：张、王、李、刘、陈、杨、赵、黄。
- 不常见姓：冉、覃、欧阳、司马、尉迟。
- 性别：男孩、女孩。
- 喜用神：金、木、水、火、土。
- 偏好：诗经、楚辞、唐诗、宋词、清雅、大气、温润、刚毅。
- 避用字：梓、轩、涵、萱、辰、煜、坤。
- 特殊输入：未知时辰、预产期模式、API 失败 fallback。

## 测试维度

自动字段：

- `knowledgeBacked`
- `validationWarnings`
- `qualityScore`
- `qualityLevel`
- `riskLevel`
- `poemSource`
- fallback 状态

人工评价维度，每个名字 1-5 分：

- 好听度
- 文化感
- 可信度
- 独特性
- 易读易记
- 生僻风险，5 分表示风险低
- 是否像 AI 编的，5 分表示不像 AI 编的
- 是否愿意收藏
- 是否愿意付费看完整报告

## 通过标准

- `knowledgeBacked` 案例占比 >= 85%。
- fallback 触发率 <= 10%，且必须显式标记。
- 避用字命中失败率 = 0。
- 编造出处风险 <= 3%，且应进入 `reject` 或明确 warning。
- 平均 `qualityScore` >= 78。
- 人工可用率 >= 70%，可用定义为人工平均分 >= 3.6。
- 不允许出现合规禁词未被 warning/reject 捕获。

## 问题汇总

| 等级 | 问题 | 影响 | 建议 |
|---|---|---|---|
| P1 | fallback 名字池未按用户避用字过滤，QA-028 原始样本命中“萱” | 用户明确避用时仍展示，信任损伤较大 | 已小修：fallback 生成时过滤 `excludeChars` |

## 后续修复建议

本阶段只建议进入 P1 小修，不建议扩功能：

1. fallback 避用字过滤已完成小修，需在下一轮真实 API 批跑中复核。
2. 将 fallback 的 `knowledgeBacked` 固定为 `false`，并保留“基础推荐”标识。
3. 为知识层继续补 20-30 条高频可用出处卡，重点覆盖“土、金、刚毅、大气”方向。
4. 增加批内去重规则，避免同一轮展示重复或高度相似名字。
5. 复姓输入单独保留人工 QA 标记，暂不做复杂音韵算法。

## 阶段结论

- 不建议直接进入 SEO / 付费前置阶段。
- P1 fallback 避用字小修已完成，建议下一步做一次真实 LLM 批量回归。
- 当前系统已具备“知识层校验 + 质量评分 + 轻量人工 QA”的基础，但文化背书覆盖率和 fallback 细节还需要再稳一轮。
